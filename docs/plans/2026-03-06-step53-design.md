## 1. 테마 중앙화 (Flutter)

### 1.1 목표
- 하드코딩된 `Colors.xxx` ~70건을 시맨틱 `AppColors` ThemeExtension으로 교체
- light/dark 모드 자동 전환 지원 (Material 3 공식 패턴)

### 1.2 AppColors ThemeExtension

`app/lib/config/theme.dart`에 추가:

```dart
@immutable
class AppColors extends ThemeExtension<AppColors> {
  final Color success;       // 적립, 가격 하락(긍정)
  final Color error;         // 삭제, 에러, 가격 상승(부정)
  final Color warning;       // 주의, 보통
  final Color info;          // 정보, 알림
  final Color neutral;       // 비활성 아이콘/텍스트
  final Color neutralLight;  // 배경, placeholder
  final Color neutralBorder; // 테두리

  // 브랜드 고정 (light/dark 동일)
  static const kakao = Color(0xFFFEE500);
  static const naver = Color(0xFF03C75A);

  const AppColors({
    required this.success,
    required this.error,
    required this.warning,
    required this.info,
    required this.neutral,
    required this.neutralLight,
    required this.neutralBorder,
  });

  @override
  AppColors copyWith({...});

  @override
  AppColors lerp(AppColors? other, double t);
}
```

## 2. price_history 보존 정책 (서버)

### 2.1 목표
- 2년 이내: 상세 데이터 보존 (기존 price_history)
- 2년 이후: 월별 집계로 압축 후 원본 파티션 DROP

### 2.2 Migration 017: price_history_monthly

```sql
CREATE TABLE price_history_monthly (
    id           BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    product_id   BIGINT NOT NULL REFERENCES products(id),
    year_month   DATE NOT NULL,
    avg_price    INTEGER NOT NULL,
    min_price    INTEGER NOT NULL,
    max_price    INTEGER NOT NULL,
    first_price  INTEGER NOT NULL,
    last_price   INTEGER NOT NULL,
    record_count INTEGER NOT NULL,
    had_stockout BOOLEAN NOT NULL DEFAULT FALSE,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (product_id, year_month)
);

CREATE INDEX idx_phm_product_month
    ON price_history_monthly (product_id, year_month DESC);
```

### 2.3 집계 + 정리 로직

`ensure_partitions()` 확장 (main.rs, 24시간 주기):

1. 파티션 생성 (기존 — 현재월 + 3개월)
2. 2년 이전 파티션 탐색 (`pg_inherits`)
3. 해당 파티션 데이터 → `price_history_monthly` 집계 INSERT (ON CONFLICT 무시)
4. 집계 행 수 검증 후 파티션 DROP

**안전장치:**
- 집계 record_count와 원본 COUNT 비교 — 불일치 시 DROP 중단
- 한 번에 1개 파티션만 처리
- tracing::info 로그

### 2.4 집계 쿼리

```sql
INSERT INTO price_history_monthly
    (product_id, year_month, avg_price, min_price, max_price,
     first_price, last_price, record_count, had_stockout)
SELECT
    product_id,
    DATE_TRUNC('month', recorded_at)::DATE,
    AVG(price)::INTEGER,
    MIN(price),
    MAX(price),
    (ARRAY_AGG(price ORDER BY recorded_at ASC))[1],
    (ARRAY_AGG(price ORDER BY recorded_at DESC))[1],
    COUNT(*),
    BOOL_OR(is_out_of_stock)
FROM price_history_YYYYMM
GROUP BY product_id, DATE_TRUNC('month', recorded_at)::DATE
ON CONFLICT (product_id, year_month) DO NOTHING;
```

### 2.5 읽기 경로 변경

- 날짜 범위 2년 이내 → `price_history` 조회 (기존)
- 2년 이전 포함 → `price_history_monthly` UNION ALL
- Flutter 측 변경 없음 (응답 형식 동일)
- `stats.rs` 30일 평균/최저가 → 변경 없음 (항상 최근 데이터)
